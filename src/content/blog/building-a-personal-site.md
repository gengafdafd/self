# 把个人站从仓库页整理成真正的网站

一开始，这个仓库只是一个 GitHub Pages 仓库，页面内容零散、目录混乱、部署方式也不统一。

后面做的第一件事，不是换主题，而是先把发布链路理顺：

- 清理重复的 Pages workflow
- 统一改成 Astro 构建 `dist/`
- 把自定义域名迁到最终产物里

只有发布链路稳定之后，视觉和内容优化才有意义。

## 为什么要迁到 Astro

因为目标已经不是“展示一个 README”，而是：

- 有首页
- 有博客列表
- 有文章详情页
- 后续还能继续扩展

这种场景下，Markdown 直接裸露成文件页明显不够。

## 这次整理的核心

这次调整的核心不是做得更花，而是让它更像一个长期可维护的网站：

- 主页承担识别和入口
- 博客承担持续更新
- 文档承担说明和沉淀

## 下一步

后续最重要的不是再换框架，而是持续补真实内容，让这个站点慢慢长出来。
